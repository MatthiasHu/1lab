Representable functors🔗

A functor F:Cop→SetsκF : \ca{C}\op \to \sets_\kappaF:Cop→Setsκ​ (from a locally κ\kappaκ-small category) is said to be representable when it is naturally isomorphic to Hom(−,X)\hom(-, X)Hom(−,X) for an object X:CX : \ca{C}X:C (called the representing object) — that is, the functor FFF classifies the maps into XXX. Note that we can evidently dualise the definition, to get what is called a corepresentable functor, one of the form Hom(X,−)\hom(X, -)Hom(X,−), but we refer informally to both of these situations as “representables” and “representing objects”.

record Representation (F : Functor (C ^op) (Sets κ)) : Type (o ⊔ κ) where
  field
    rep        : C.Ob
    represents : F C^.≅ よ₀ C rep

open Representation

This definition is deceptively simple: the idea of representable functor (and of representing object) is key to understanding the idea of universal property, which could be called the most important concept in category theory. Most constructions in category theory specified in terms of the existence of certain maps are really instances of representing objects for functors: limits, colimits, coends, adjoint functors, Kan extensions, etc.

The first thing we will observe is an immediate consequence of the Yoneda lemma: representing objects are unique. Intuitively this is because “XXX is a representation of FFF” determines how XXX reacts to being mapped into, and since the only thing we can probe objects in an arbitrary category by are morphisms, two objects which react to morphisms in the same way must be isomorphic.

representation-unique : {F : Functor (C ^op) (Sets κ)} (X Y : Representation F)
                      → X .rep C.≅ Y .rep
representation-unique X Y =
  is-ff→essentially-injective {F = よ C} (よ-is-fully-faithful C) よX≅よY
  where
    よX≅よY : よ₀ C (X .rep) C^.≅ よ₀ C (Y .rep)
    よX≅よY = (X .represents C^.Iso⁻¹) C^.∘Iso Y .represents

As initial objects🔗

We begin to connect the idea of representing objects to other universal constructions by proving this alternative characterisation of representations: A functor FFF is representable if, and only if, its category of elements ∫F\int F∫F has a terminal object.

terminal-element→representation
  : {F : Functor (C ^op) (Sets κ)} → Terminal (∫ C F) → Representation F
terminal-element→representation {F} term = f-rep where
  module F = Functor F
  open Terminal term

From the terminal object in ∫F\int F∫F1, we obtain a natural transformation ηy:F(y)→Hom(y,X)\eta_y : F(y) \to \hom(y,X)ηy​:F(y)→Hom(y,X), given componentwise by interpreting each pair (y,s)(y, s)(y,s) as an object of ∫F\int F∫F, then taking the terminating morphism (y,s)→(X,F(X))(y, s) \to (X, F(X))(y,s)→(X,F(X)), which satisfies (by definition) F(!)(F(X))=sF(!)(F(X)) = sF(!)(F(X))=s. This natural transformation is componentwise invertible, as the calculation below shows, so it constitutes a natural isomorphism.

  nat : F => よ₀ C (top .ob)
  nat .η ob section = has⊤ (elem ob section) .centre .hom
  nat .is-natural x y f = funext λ sect → ap hom $ has⊤ _ .paths $ elem-hom _ $
    F.₁ (has⊤ _ .centre .hom C.∘ f) (top .section)   ≡⟨ happly (F.F-∘ _ _) _ ⟩≡
    F.₁ f (F.₁ (has⊤ _ .centre .hom) (top .section)) ≡⟨ ap (F.₁ f) (has⊤ _ .centre .commute) ⟩≡
    F.₁ f sect                                       ∎

  inv : ∀ x → Sets.is-invertible (nat .η x)
  inv x = Sets.make-invertible
    (λ f → F.₁ f (top .section))
    (funext λ x → ap hom $ has⊤ _ .paths (elem-hom x refl))
    (funext λ x → has⊤ _ .centre .commute)

  f-rep : Representation F
  f-rep .rep = top .ob
  f-rep .represents = C^.invertible→iso nat $
    componentwise-invertible→invertible nat inv

Universal constructions🔗

We now show a partial converse to the calculation above: That terminal objects are representing objects for a particular functor. Consider, to be more specific, the constant functor F:Cop→SetsF : \ca{C}\op \to \setsF:Cop→Sets which sends everything to the terminal set. When is FFF representable?

Well, unfolding the definition, it’s when we have an object X:CX : \ca{C}X:C with a natural isomorphism Hom(−,X)≅F\hom(-,X) \cong FHom(−,X)≅F. Unfolding that, it’s an object XXX for which, given any other object YYY, we have an isomorphism of sets Hom(Y,X)≅{∗}\hom(Y,X) \cong \{*\}Hom(Y,X)≅{∗}2. Hence, a representing object for the “constantly {∗}\{*\}{∗}” functor is precisely a terminal object. It turns out the
