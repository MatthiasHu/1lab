The bicategory of spans🔗

Let C\ca{C}C be a precategory. By a span in C\ca{C}C (from an object A:CA : \ca{C}A:C to an object B:CB : \ca{C}B:C), we mean a diagram of the form A←C→BA \ot C \to BA←C→B. Note that the “vertex” of this span, the object CCC, is part of the data, so that the collection of “spans in C\ca{C}C” will not be a set (unless C\ca{C}C is strict) — so we can not construct a category where Hom(a,b)\hom(a,b)Hom(a,b) is the collection of spans from aaa to bbb.

However, we can make spans in C\ca{C}C the objects of a category, and the hom-sets are the maps in C\ca{C}C between the vertices which “commute with the legs”. Diagramatically, a map between spans is the dashed line in

where both the left and right triangles must commute.

record Span (a b : Ob) : Type (o ⊔ ℓ) where
  constructor span
  field
    apex  : Ob
    left  : Hom apex a
    right : Hom apex b

open Span

record Span-hom {a b : Ob} (x y : Span a b) : Type ℓ where
  no-eta-equality
  constructor span-hom
  field
    map   : Hom (x .apex) (y .apex)
    left  : x .left  ≡ y .left ∘ map
    right : x .right ≡ y .right ∘ map

The category SpansC(A,B)\id{Spans}_\ca{C}(A, B)SpansC​(A,B) of spans between AAA and BBB admits a faithful functor to C\ca{C}C (projecting the vertex and the “middle map”, respectively), so that equality of maps of spans can be compared at the level of maps in C\ca{C}C.

Spans : Ob → Ob → Precategory _ _
Spans x y .Precategory.Ob = Span x y
Spans x y .Precategory.Hom = Span-hom
Spans x y .Precategory.Hom-set _ _ =
  is-hlevel≃ 2 (Iso→Equiv eqv e⁻¹) (hlevel 2)
Spans x y .Precategory.id = span-hom id (intror refl) (intror refl)
Spans x y .Precategory._∘_ f g =
  span-hom (f .map ∘ g .map)
    (g .left ∙ pushl (f .left))
    (g .right ∙ pushl (f .right))
Spans x y .Precategory.idr f = Span-hom-path (idr _)
Spans x y .Precategory.idl f = Span-hom-path (idl _)
Spans x y .Precategory.assoc f g h = Span-hom-path (assoc _ _ _)

Now suppose that C\ca{C}C admits pullbacks for arbitrary pairs of maps. Supposing that we have some spans S:A→BS : A \to BS:A→B and S′:B→CS' : B \to CS′:B→C, we can fit them in an M-shaped diagram like

so that taking the pullback of the diagram S→B←S′S \to B \ot S'S→B←S′ gives us an universal solution to the problem of finding a span A←(S×BS′)→CA \ot (S \times_B S') \to CA←(S×B​S′)→C. Since pullbacks are universal, this composition operation is functorial, and so extends to a composition operation Span-∘:

module _ (pb : ∀ {a b c} (f : Hom a b) (g : Hom c b) → Pullback C f g) where
  open Functor

  Span-∘ : ∀ {a b c} → Functor (Spans b c ×ᶜ Spans a b) (Spans a c)
  Span-∘ .F₀ (sp1 , sp2) =
    span pb.apex (sp2 .left ∘ pb.p₂) (sp1 .right ∘ pb.p₁)
    where module pb = Pullback (pb (sp1 .left) (sp2 .right))

  Span-∘ .F₁ {x1 , x2} {y1 , y2} (f , g) = res
    where
      module x = Pullback (pb (x1 .left) (x2 .right))
      module y = Pullback (pb (y1 .left) (y2 .right))

      x→y : Hom x.apex y.apex
      x→y = y.limiting {p₁' = f .map ∘ x.p₁} {p₂' = g .map ∘ x.p₂} p
        where abstract
          open Pullback
          p : y1 .left ∘ f .map ∘ x.p₁ ≡ y2 .right ∘ g .map ∘ x.p₂
          p = pulll (sym (f .left)) ∙ x.square ∙ pushl (g .right)

      res : Span-hom _ _
      res .map = x→y
      res .left = sym (pullr y.p₂∘limiting ∙ pulll (sym (g .left)))
      res .right = sym (pullr y.p₁∘limiting ∙ pulll (sym (f .right)))

  Span-∘ .F-id {x1 , x2} = Span-hom-path $ sym $ x.unique id-comm id-comm
    where module x = Pullback (pb (x1 .left) (x2 .right))

  Span-∘ .F-∘ {x1 , x2} {y1 , y2} {z1 , z2} f g =
    Span-hom-path $ sym $ z.unique
      (pulll z.p₁∘limiting ∙ pullr y.p₁∘limiting ∙ assoc _ _ _)
      (pulll z.p₂∘limiting ∙ pullr y.p₂∘limiting ∙ assoc _ _ _)
    where
      module x = Pullback (pb (x1 .left) (x2 .right))
      module y = Pullback (pb (y1 .left) (y2 .right))
      module z = Pullback (pb (z1 .left) (z2 .right))

What we’ll show in the rest of this module is that Span-∘ lets us make Spans into the categories of 1-cells of a prebicategory, the (pre)bicategory of spans (of C\ca{C}C) Span(C)\id{Span}(\ca{C})Span(C). As mentioned before, this prebicategory has (a priori) no upper bound on the h-levels of its 1-cells, so it is not locally strict. We remark that when C\ca{C}C is univalent, then Span(C)\id{Span}(\ca{C})Span(C) is locally so, and when C\ca{C}C is gaunt, then Span(C)\id{Span}(\ca{C})Span(C) is strict.
